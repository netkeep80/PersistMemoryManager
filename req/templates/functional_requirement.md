# Шаблон: functional requirement (`fr-xxx`)

Функциональное требование описывает поведение системы при определённых
условиях. Размещается в [`05_functional_requirements.md`](../05_functional_requirements.md).
Идентификатор: `fr-xxx`.

## Обязательные поля

| Поле                          | Назначение                                                       |
|-------------------------------|-------------------------------------------------------------------|
| `Тип`                         | Должно быть `functional requirement`.                            |
| `Название`                    | Короткое русскоязычное название.                                 |
| `Приоритет`                   | `Must` / `Should` / `Could` / `Won't`.                            |
| `Статус`                      | `Draft` / `Active` / `Recovered` / `Deprecated` / `Superseded`.   |
| `Источник`                    | Документ или анкер исходного кода, основание для требования.     |
| `Формулировка`                | Подробное русскоязычное описание функционального поведения.       |
| `Триггер/условие`             | Что инициирует поведение (вызов API, событие, фаза lifecycle).    |
| `Поведение системы`           | Что делает система в ответ на триггер.                            |
| `Результат`                   | Постусловие/изменения состояния и возвращаемые значения.          |
| `Ошибки/отказы`               | Какие ошибки/исключения возможны, как сигнализируются.            |
| `Контекст и обоснование`      | Почему требование существует, какие альтернативы рассматривались. |
| `Реализует`                   | Ссылки на `feat-*`, `ur-*`, `br-*`, которые требование реализует. |
| `Реализуется в`               | Ссылки на PMM anchors в `include/pmm/**`.                          |
| `Проверяется в`               | Ссылки на acceptance criteria и/или тесты.                        |
| `Примечания` *(опц.)*         | Свободные пояснения, ссылки на discussions.                       |

## Правила трассировки

- `Реализуется в` обязательно содержит хотя бы одну ссылку на PMM anchor либо
  на дочернее FR. PMM anchor должен содержать обратную `req:`-аннотацию на
  это требование.
- Для `Must`/`Should` обязательно `Проверяется в` с минимум одним AC или
  тестом.

## Корректный пример

```md
## fr-002

**Тип:** functional requirement
**Название:** Загрузка существующего persistent address space
**Приоритет:** Must
**Статус:** Active
**Источник:** README API, обзор `include/pmm/persist_memory_manager.h`

**Формулировка:**
Менеджер памяти должен предоставлять операцию загрузки уже существующего
persistent address space через публичный API `load(VerifyResult&)`. Операция
должна проверять метаданные образа, диагностировать повреждения, восстанавливать
служебные индексы там, где это предусмотрено моделью восстановления, и
возвращать результат загрузки через `VerifyResult`.

**Триггер/условие:** Вызов `PersistMemoryManager::load(VerifyResult&)` после
открытия persistent storage backend.

**Поведение системы:**
1. Считывает заголовок `ManagerHeader` из `Block_0`.
2. Проверяет magic, version, layout-специфичные инварианты.
3. Восстанавливает linked list блоков, counters и free tree.
4. Заполняет `VerifyResult` диагностикой.

**Результат:** Менеджер находится в согласованном состоянии, либо вернул
структурированную ошибку.

**Ошибки/отказы:**
- неподдерживаемая `image_version` → диагностика unsupported format;
- повреждённый header → диагностика header corruption и отказ от repair.

**Контекст и обоснование:**
Загрузка является ключевым use case PMM, поскольку библиотека управляет
persistent address space, переживающим перезапуск.

**Реализует:**

- [ur-005](../03_user_requirements.md#ur-005)
- [feat-001](../04_features.md#feat-001)
- [feat-004](../04_features.md#feat-004)

**Реализуется в:**

- [pmm-persistmemorymanager-load](../../include/pmm/persist_memory_manager.h#pmm-persistmemorymanager-load)

**Проверяется в:**

- [ac-006](../12_acceptance_criteria.md#ac-006)

**Примечания:**
`verify()` остаётся read-only диагностикой; `load(VerifyResult&)` может
выполнять документированное восстановление служебных структур.
```

## Некорректные примеры

```md
## fr-099
- Требование: load работает.
- Source: include/pmm/persist_memory_manager.h — анкер pmm-persistmemorymanager-load
```

Ошибки:

- ссылка на anchor оформлена текстом, а не Markdown-ссылкой;
- отсутствуют все обязательные поля и структура.
