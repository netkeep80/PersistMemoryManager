# Шаблон: dependency (`dep-xxx`)

Dependency фиксирует внешнюю или внутреннюю зависимость PMM от другой
подсистемы, библиотеки или инструмента. Размещается в
[`11_assumptions_dependencies.md`](../11_assumptions_dependencies.md).
Идентификатор: `dep-xxx`.

## Обязательные поля

| Поле                          | Назначение                                                       |
|-------------------------------|-------------------------------------------------------------------|
| `Тип`                         | Должно быть `dependency`.                                        |
| `Название`                    | Короткое русскоязычное название зависимости.                     |
| `Приоритет`                   | `Must` / `Should` / `Could` / `Won't`.                            |
| `Статус`                      | `Draft` / `Active` / `Recovered` / `Deprecated` / `Superseded`.   |
| `Источник`                    | Документ или артефакт.                                            |
| `Формулировка`                | Подробное русскоязычное описание зависимости.                     |
| `Зависимость`                 | Что является зависимостью (библиотека, утилита, стандарт).       |
| `Тип зависимости`             | `build` / `runtime` / `test` / `tooling`.                         |
| `Версия/условие`              | Минимальная/допустимая версия и feature-flags.                   |
| `Влияние на build/runtime`    | Как зависимость встраивается и что произойдёт при её отсутствии.  |
| `Fallback/mitigation`         | План при отсутствии или несовместимости (или `нет`).              |
| `Контекст и обоснование`      | Почему зависимость нужна.                                         |
| `Связано с`                   | Ссылки на FR/feat/sys, использующие зависимость.                  |
| `Реализуется в` *(опц.)*      | CMake-таргеты, find_package, include-блоки.                      |
| `Проверяется в` *(опц.)*      | Acceptance criteria или CI job.                                   |
| `Примечания` *(опц.)*         | Свободные пояснения.                                              |

## Правила трассировки

- `Связано с` обязательно для `Active`/`Recovered`.
- Если зависимость опциональна — явно указать условия включения.

## Корректный пример

```md
## dep-001

**Тип:** dependency
**Название:** Catch2 для unit-тестов
**Приоритет:** Must
**Статус:** Active
**Источник:** [docs/test_matrix.md](../docs/test_matrix.md)

**Формулировка:**
Тесты PMM используют фреймворк Catch2 для unit-тестирования. Зависимость
является test-only и не присутствует в публичной поверхности библиотеки.

**Зависимость:** Catch2.

**Тип зависимости:** test.

**Версия/условие:** Catch2 v3.x; подключается через CMake `FetchContent` или
package manager.

**Влияние на build/runtime:** при отсутствии Catch2 не собирается test target,
основная сборка не затрагивается.

**Fallback/mitigation:** возможна сборка без тестов через CMake флаг
`PMM_BUILD_TESTS=OFF`.

**Контекст и обоснование:**
Catch2 даёт компактные test cases, header-only тестовый фреймворк, удобный
для kernel-библиотеки.

**Связано с:**

- [sys-001](../10_system_requirements.md#sys-001)
- [feat-009](../04_features.md#feat-009)
```

## Некорректные примеры

- Не указан `Тип зависимости` — неясно, влияет ли на runtime пользователя.
- Нет `Fallback/mitigation`, хотя зависимость опциональна.
