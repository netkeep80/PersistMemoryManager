# Шаблон: assumption (`asm-xxx`)

Assumption фиксирует явное предположение, на котором строится дизайн или
реализация PMM. Размещается в
[`11_assumptions_dependencies.md`](../11_assumptions_dependencies.md).
Идентификатор: `asm-xxx`.

## Обязательные поля

| Поле                          | Назначение                                                       |
|-------------------------------|-------------------------------------------------------------------|
| `Тип`                         | Должно быть `assumption`.                                        |
| `Название`                    | Короткое русскоязычное название.                                 |
| `Приоритет`                   | `Must` / `Should` / `Could` / `Won't`.                            |
| `Статус`                      | `Draft` / `Active` / `Recovered` / `Deprecated` / `Superseded`.   |
| `Источник`                    | Документ или артефакт.                                            |
| `Формулировка`                | Подробное русскоязычное описание предположения.                   |
| `Предположение`               | Что именно мы считаем истинным.                                   |
| `Где используется`            | Какие подсистемы/требования полагаются на предположение.          |
| `Риск при нарушении`          | Что произойдёт, если предположение перестанет выполняться.        |
| `Fallback/mitigation`         | План на случай нарушения предположения (или `нет`).               |
| `Контекст и обоснование`      | Почему предположение принято.                                     |
| `Связано с`                   | Ссылки на FR/feat/dr, опирающиеся на предположение.               |
| `Реализуется в` *(опц.)*      | PMM anchors, ASSERT'ы или тесты, фиксирующие предположение.       |
| `Проверяется в` *(опц.)*      | Acceptance criteria или тесты.                                    |
| `Примечания` *(опц.)*         | Свободные пояснения, ссылки на discussions.                       |

## Правила трассировки

- `Связано с` обязательно для `Active`/`Recovered`.
- Если предположение фиксируется ASSERT/static_assert — указать anchor.

## Корректный пример

```md
## asm-002

**Тип:** assumption
**Название:** Стабильное выравнивание persistent аллокаций
**Приоритет:** Must
**Статус:** Active
**Источник:** [docs/internal_structure.md](../docs/internal_structure.md)

**Формулировка:**
Все persistent аллокации выровнены по `granule` storage backend'а; адресная
модель `pptr` опирается на это выравнивание.

**Предположение:** backend гарантирует, что `data()` выровнен на `granule`.

**Где используется:** address traits, allocator, pptr conversions.

**Риск при нарушении:** некорректные `pptr` ↔ raw указатели → silent
corruption.

**Fallback/mitigation:** runtime-проверка выравнивания при `create`/`load`,
отказ от инициализации с диагностикой.

**Контекст и обоснование:**
Все стандартные backend'ы PMM соблюдают это выравнивание; пользовательские
backend'ы должны соответствовать concept'у.

**Связано с:**

- [dr-007](../06_data_requirements.md#dr-007)
- [qa-mem-001](../08_quality_attributes.md#qa-mem-001)
```

## Некорректные примеры

- Нет поля `Риск при нарушении` — невозможно оценить blast radius.
- Поле `Fallback/mitigation` опущено и нет указания "нет" — неясно, есть ли план.
