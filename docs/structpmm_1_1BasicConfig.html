Detailed Description

template<typename AddressTraitsT = DefaultAddressTraits, typename LockPolicyT = config::NoLock, std::size_t GrowNum = config::kDefaultGrowNumerator, std::size_t GrowDen = config::kDefaultGrowDenominator, std::size_t MaxMemoryGB = 64, typename LoggingPolicyT = logging::NoLogging>
struct pmm::BasicConfig< AddressTraitsT, LockPolicyT, GrowNum, GrowDen, MaxMemoryGB, LoggingPolicyT >

Базовый шаблон конфигурации менеджера с HeapStorage (Issue #160).

Устраняет дублирование между CacheManagerConfig, PersistentDataConfig, EmbeddedManagerConfig, IndustrialDBConfig и LargeDBConfig. Готовые конфигурации теперь являются псевдонимами BasicConfig с конкретными параметрами.

Template Parameters
AddressTraitsTТип адресного пространства (DefaultAddressTraits, LargeAddressTraits, etc.)
LockPolicyTПолитика блокировок (config::NoLock или config::SharedMutexLock)
GrowNumЧислитель коэффициента роста хранилища (по умолчанию 5)
GrowDenЗнаменатель коэффициента роста хранилища (по умолчанию 4, т.е. рост 25%)
MaxMemoryGBМаксимальный объём памяти в ГБ (0 = без ограничения)
LoggingPolicyTПолитика логирования (logging::NoLogging по умолчанию) (Issue #202, Phase 4.2)

Пример создания собственной конфигурации:

// Многопоточный менеджер с 50% ростом и 32 ГБ лимитом
using MyConfig = pmm::BasicConfig<
pmm::DefaultAddressTraits,
pmm::config::SharedMutexLock,
3, 2, // grow 3/2 = 50%
32 // max 32 GB
>;
using MyManager = pmm::PersistMemoryManager<MyConfig>;
// Менеджер с логированием в stderr
using DebugConfig = pmm::BasicConfig<
pmm::DefaultAddressTraits,
pmm::config::NoLock,
5, 4, 64,
pmm::logging::StderrLogging
>;
