Detailed Description

template<typename ManagerT>
struct pmm::pstringview< ManagerT >

Персистентная интернированная read-only строка (Issue #151, #184).

Хранит длину и строковые данные непосредственно в блоке (Issue #184). Оптимизация: вместо двух блоков (pstringview + char[]) используется один блок, содержащий длину и строку. Это существенно экономит память и ускоряет работу pmap<pptr<pstringview>, _Tvalue>.

Объекты pstringview живут в ПАП и не могут быть созданы на стеке напрямую.

Простой API (рекомендуемый способ):

// Конструктор-хелпер: создаёт временный объект, возвращает pptr через implicit conversion
Mgr::pptr<Mgr::pstringview> p = Mgr::pstringview("hello");
Mgr::pptr<Mgr::pstringview> p2 = Mgr::pstringview("hello");
assert(p == p2); // true — дедупликация

AVL-дерево: каждый pstringview использует встроенные поля TreeNode своего блока (left_offset, right_offset, parent_offset, avl_height) как ссылки AVL-дерева словаря интернирования. Это является частью "леса AVL-деревьев" ПАП.

Инварианты:

  • str[] содержит null-terminated строку, расположенную сразу за полем length.
  • Два pstringview с одинаковым содержимым — это один объект (один granule-индекс).

static psview_pptr pmm::pstringview< ManagerT >::intern ( const char *  s)
inlinestaticnoexcept

Интернировать строку s: найти существующий pstringview или создать новый.

Выполняет поиск в AVL-дереве по лексикографическому ключу. Если строка найдена — возвращает существующий pptr. Если нет — создаёт новый pstringview-блок, блокирует его навечно и вставляет в AVL-дерево.

Parameters
sC-строка для интернирования (nullptr обрабатывается как "").
Returns
pptr<pstringview<ManagerT>> — персистентный указатель на pstringview. Нулевой pptr при ошибке аллокации.

Равенство двух pstringview.
